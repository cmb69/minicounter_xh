Minicounter_XH - a CMSimple_XH Plugin

Minicounter_XH facilitates counting the visitors of your website. It's a very minimalistic counter, that will only count the total number of visitors, but this could be implemented in a very efficient manner, so it's blazingly fast. Contrary to many other counters Minicounter_XH is session based.

Requirements

Minicounter_XH is a plugin for CMSimple_XH. It requires a UTF-8 encoded version.

Installation

The installation is done as with many other CMSimple_XH plugins. See the CMSimple_XH wiki for further details.

  1. Backup the data on your server.
  2. Unzip the distribution on your computer.
  3. Upload the whole directory minicounter/ to your server to CMSimple_XH's plugins directory.
  4. Set write permissions to the subdirectories config/, css/ and languages/.
  5. Switch to "Minicounter" in the back-end to check if all requirements are fulfilled.

Note that the minicounter data file could be accessed directly by calling its URL. A .htaccess file to prohibit this is already contained in the default data folder. For other servers or for custom data folders you should take care for yourself, that this file is protected from non authorized access.

Settings

The plugin's configuration is done as with many other CMSimple_XH plugins in the website's back-end. Select "Minicounter" from "Plugins".

You can change the default settings of Minicounter_XH in "Config". Hints for the options will be displayed when hovering over the help icon with your mouse.

Localization is done in "Language". You can translate the character strings to your own language, or customize them according to your needs.

Usage

To display the counter on your website insert on a CMSimple page:

{{{PLUGIN:minicounter();}}}

or in the template:

<?php echo minicounter();?>

You can change the displayed text in the plugin back-end under "Language". Please note the text was chosen this way, as Minicounter_XH doesn't display the actual visitor count, but insteads keeps a fixed number for every visitor. You can see the effect, if you surf your site with two different browsers.

Visits in admin mode (including login and logout) are not counted. Furthermore it's possible to exclude your IP address completely in the configuration.

There's no setting how long a visitor may be inactive and still be counted as the same visitor. All that happens transparently through the use of sessions. As long as the session is active, the visitor will be regarded the same. When the session will be closed, depends on the session.gc_* PHP settings, and on the plugin that initially started the session (usually the session ends at the latest, when the browser is closed).

Limitations

Minicounter_XH is not accurate! In fact it is not possible to accurately count visitors (contrary to page hits) on a website at all. That stems from the fact that in general it's impossible to identify a visitor reliably. From call to call his IP address may change, if he sits behind a proxy. And over time IPv4 addresses will change anyway. The session based approach used by Minicounter_XH is perhaps even more inaccurate, as it will work only, if the session is accepted by the visitor (typically by accepting the session cookie). To avoid counting visitors, who don't accept sessions, on every single page hit, Minicounter_XH counts only those who do, what's possible only, if they call at least two pages. IOW: visitors hitting only once won't be counted either.
