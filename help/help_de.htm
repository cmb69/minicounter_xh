Minicounter_XH – ein CMSimple_XH Plugin

Minicounter_XH ermöglicht es, die Besucher Ihrer Website zu zählen. Es ist ein sehr minimalistischer Zähler, der nur die Gesamtanzahl der Besucher zählt, aber dies konnte in einer sehr effizienten Weise implementiert werden, indem die nötigen Information temporär in einer PHP Sitzung gespeichert werden.

Voraussetzungen

Minicounter_XH ist ein Plugin für CMSimple_XH. Es benötigt eine UTF-8 kodierte Version.

Installation

Die Installation erfolgt wie bei vielen anderen CMSimple_XH-Plugins auch. Im CMSimple_XH-Wiki finden Sie weitere Details.

  1. Sichern Sie die Daten auf Ihrem Server.
  2. Entpacken Sie die ZIP-Datei auf Ihrem Rechner.
  3. Laden Sie das ganze Verzeichnis minicounter/ auf Ihren Server in CMSimple_XHs Plugin-Verzeichnis hoch.
  4. Machen Sie die Unterverzeichnisse config/ und languages/ sowie den eingestellten Datenordner beschreibbar.
  5. Gehen Sie zu "Minicounter" im Administrationsbereich, um zu prüfen, ob alle Voraussetzungen erfüllt sind.

Beachten Sie, dass auf die Minicounter Zählerdatei durch Aufruf ihrer URL direkt zugegriffen werden kann. Eine .htaccess Datei um das zu verhinden, befindet sich bereits im voreingestellten Datenordner. Für andere Server oder benutzerdefinierte Datenordner sollten Sie selbst sicher stellen, dass diese Datei vor unberechtigtem Zugriff geschützt ist.

Einstellungen

Die Plugin-Konfiguration erfolgt wie bei vielen anderen CMSimple_XH-Plugins auch im Administrationsbereich der Website. Wählen Sie "Plugins" → "Minicounter".

Sie können die Voreinstellungen von Minicounter_XH unter "Konfiguration" ändern. Hinweise zu den Optionen werden beim Überfahren der Hilfe-Icons mit der Maus angezeigt.

Die Lokalisierung wird unter "Sprache" vorgenommen. Sie können die Sprachtexte in Ihre eigene Sprache übersetzen, oder diese Ihren Wünschen gemäß anpassen.

Verwendung

Um den Zähler auf Ihrer Website anzuzeigen fügen Sie auf einer CMSimple-Seite ein:

    {{{PLUGIN:minicounter();}}}

oder im Template:

    <?php echo minicounter();?>

Sie können den angezeigten Text in der Plugin-Administration unter "Sprache" ändern. Bitte beachten Sie, dass der Text so gewählt wurde, da Minicounter_XH nicht die eigentliche Besucherzahl anzeigt, sondern eine feste Ordinalzahl für jeden Besucher vergibt. Sie können diesen Effekt sehen, wenn Sie Ihre Website mit zwei verschiedenen Browsern aufrufen.

Besuche im Adminmodus (einschließlich Login und Logout) werden nicht gezählt. Darüber hinaus ist es möglich IP-Adressen vollständig per Konfigurationsoption auszuschließen.

Es gibt keine Einstellung für die maximale Inaktivitätsdauer eines Besuchers, so dass er weiterhin als der selbe Besucher angesehen wird. Dies wird automatisch durch die Verwendung von Sitzungen geregelt. So lange die Sitzung aktiv ist, wird der Besucher als der selbe angesehen. Wann die Sitzung geschlossen wird, hängt von den session.gc_* PHP-Einstellungen ab, und von dem Plugin, das die Sitzung initialisiert hat (normalerweise ended die Sitzung spätestens, wenn der Browser geschlossen wird).

Einschränkungen

  • Die Verwendung von Besucher-Trackern kann aus Datenschutzgründen durch Gesetze eingeschränkt oder gar ganz verboten sein. Sie können Minicounter_XH so konfigurieren, dass das DNT (Do Not Track) Flag berücksichtigt wird, was ausreichen könnte. Im Zweifel sollten Sie einen Anwalt befragen.

  • Minicounter_XH ist nicht genau! In der Tat ist es grundsätzlich nicht möglich Besucher einer Website genau zu zählen (im Ggs. zu Seitenaufrufen). Das kommt daher, dass es im allgemeinen unmöglich ist einen Besucher zuverlässig zu identifizieren. Von Aufruf zu Aufruf kann sich seine IP-Adresse ändern, wenn er sich hinter einem Proxy befindet. Und im Laufe der Zeit ändern sich IPv4 Adressen sowieso. Andere Techniken wie Browser-Fingerprinting und Tracking-Bilder sind ebenfalls nicht zu 100% verlässlich.

    Der Sizungsbasierte Ansatz, den Minicounter_XH verwendet, ist vermutlich noch ungenauer, da er nur funktioniert, wenn die Sitzung vom Besucher akzeptiert wird (normalerweise durch Akzeptieren des Sitzungs-Cookies).
